# План выполнения курсового проекта на тему "Разработка автоматизированной системы управления образовательными процессами"

1 неделя - инициализация и постановка задачи, описание предметной области, входные/выходные данные, спец часть, утверждение темы КП, список всех сущностей и атрибутов (вся информация которая должна храниться в бд),
черновой вариант словаря данных, каркас гит-репозитория, включая readme-файл с планом работы.

2 неделя - логическое проектирование базы данных (ERD, даталогическая), нормализация БД, план индексов, словарь данных, поля, требования к ним, план миграции первая версия.

3 неделя - физическая база данных и базовая серверная логика, набор валидных(!) тестовых данных, базовые CRUD-операции и view (представления),
структурная схема, функциональная схема, первые действия миграций.

4 неделя - серверная логика, процедуры, триггеры и транзакции с обработкой ошибок, полный пакет sql-запросов, описание всех контрактов и кодов ошибок для api.

5 неделя - безопасность и администрирование, разграничение прав доступа, хэширование и шифрование, логирование пользовательских действий, скрипт и бэкапы (восстановление и резервное копирование).

6 неделя - разработка api и интеграция его с системой, аутентификация, роли, адаптивность минимум под три устройства, реализовать api, импорт-экспорт данных (в форматах csv, sql, json..),
схема интерфейса и руководство пользователя.

7 неделя - аналитика и тестирование, подсистема или модуль с диаграммами и отчетами, полный цикл тестирования, функциональное, интеграционное (апи), нагрузочное-безопасность(sql-инъекции), отказоустойчивость.

8 неделя - итог и полный рабочий проект, а также его презентация.
